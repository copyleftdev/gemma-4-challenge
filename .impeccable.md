# Design Context

Target audience: healthcare compliance reviewers, clinical AI governance teams,
security reviewers, and technical evaluators for regulated AI workflows.

Use cases: inspect why an AI-assisted healthcare navigation decision happened,
verify which signed Aion policy artifacts were active, see which rule path won,
compare competing rule matches, and fingerprint the exact model request and
response without trusting model self-explanation.

Brand personality/tone: clinical, forensic, precise, defensible, calm under
audit pressure. The interface should feel like a medical evidence bench rather
than a chatbot or marketing site.
